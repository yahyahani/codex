# Codebase Q&A

A CLI tool that indexes a codebase and lets you ask questions about it.
Answers are grounded in real code fragments using RAG (Retrieval-Augmented Generation):
the tool finds the most relevant chunks via vector similarity, then passes them to
Claude as context so the answer cites actual files and line numbers instead of guessing.

---

## Quickstart (no API key needed)

The `--mock` flag is a built-in demo mode: it runs the full pipeline
(file ingestion → chunking → embeddings → vector search) but skips the Claude API
call and prints a fixed response instead. No `ANTHROPIC_API_KEY` required.

```bash
# 1. Clone and enter the repo
git clone <repo-url>
cd codebase-qa

# 2. Copy the env template (leave it as-is for --mock mode)
cp .env.example .env

# 3. Build the Docker image (~3 min first time, cached after)
docker compose build

# 4. Index the bundled demo repo
docker compose run --rm app index /repo

# 5. Ask a question in demo mode
docker compose run --rm app ask --mock "What does the greet function do?"
```

**Example output:**

```
[MOCK] Pipeline OK — retrieval worked, Claude API call skipped.
[MOCK] In production this answer would be generated by claude-sonnet-4-6.

Sources:
  /repo/hello.py:4-6
  /repo/hello.py:9-11
```

The Sources section shows which file and line range the retrieval considered most
relevant — that part of the pipeline is fully real even in `--mock` mode.

## Using your own codebase

Point the `index` command at any local directory:

```bash
docker compose run --rm app index /absolute/path/to/your/repo
```

Or edit the `volumes` section in `docker-compose.yml` to mount a different path
at `/repo`, then use `/repo` as the argument.

---

## Real answers with a Claude API key

To get actual Claude-generated answers instead of the `[MOCK]` placeholder:

1. Get an API key at [console.anthropic.com](https://console.anthropic.com) → API Keys
2. Add it to `.env`:

```bash
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env
```

3. Run without `--mock`:

```bash
docker compose run --rm app ask "What does the greet function do?"
```

Claude will stream its answer directly to the terminal, followed by the source
references (file + line range) it drew from.

## Architecture

```
[Repo] → [Chunking] → [Embedding] → [Vector Store] → [Retrieval] → [LLM answer]
```

| Step | Implementation |
|---|---|
| File ingestion | Extension filter + ignore list (no `.git`, `node_modules`, etc.) |
| Chunking | tree-sitter for Python (per function/class); fixed-size overlapping windows for all other extensions |
| Embeddings | `sentence-transformers/all-MiniLM-L6-v2` (384-dim, L2-normalised) |
| Vector store | ChromaDB (local, no server needed) |
| LLM | Claude `claude-sonnet-4-6` via Anthropic API |
| CLI | Typer |

## Dashboard (React/Vite)

A two-column dark-theme dashboard built with React, Vite, and Tailwind CSS.
It streams answers via SSE and shows source evidence cards with the actual code
content and line numbering — like a mini editor per retrieved chunk.

**Local development (two terminals):**

```bash
# Terminal 1 — FastAPI backend
pip install -e ".[api]"
codebase-qa index examples/demo_repo
uvicorn codebase_qa.api:app --port 8000 --reload

# Terminal 2 — Vite dev server (proxies /ask → port 8000)
cd frontend
npm install
npm run dev
```

Then open **http://localhost:5173** in your browser.
Enable **Mock mode** in the UI to run without an API key.

**Docker:**

```bash
docker compose up web   # builds Node+Python, serves at http://localhost:8000
```

The Dockerfile uses a multi-stage build: a Node 20 stage compiles the React app
(`npm run build`), and the Python stage copies the resulting `dist/` to serve
via FastAPI's `StaticFiles`.

## FastAPI endpoint (optional)

An HTTP interface is included for integrations that need a web API instead of
the CLI. It adds two endpoints on top of the same pipeline:

| Endpoint | Description |
|---|---|
| `GET /health` | Health check |
| `POST /ask` | Full answer + sources in one JSON response |
| `POST /ask/stream` | Server-Sent Events stream — text chunks followed by sources |

**Run locally:**

```bash
pip install -e ".[api]"
uvicorn codebase_qa.api:app --reload
```

**Example — non-streaming:**

```bash
curl -s -X POST http://localhost:8000/ask \
     -H 'Content-Type: application/json' \
     -d '{"question": "How does the Calculator class track history?", "mock": true}' \
     | python3 -m json.tool
```

**Example — streaming (SSE):**

```bash
curl -N -X POST http://localhost:8000/ask/stream \
     -H 'Content-Type: application/json' \
     -d '{"question": "What does slugify do?", "mock": true}'
```

Each SSE event is a JSON object with `type` (`"text"` or `"sources"`) and
`content`. The stream closes with `data: [DONE]`.
